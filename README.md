# Bourdon Drone PWA

Mobile-first progressive web app for continuous bourdon / drone reference in tuning, intonation, and rehearsal sessions.

## Stack

- React + TypeScript + Vite
- Tailwind CSS utility system
- Zustand for app state and preset persistence
- Web Audio API (custom synthesis engine)
- `vite-plugin-pwa` for manifest + service worker registration

## Run

```bash
npm install
npm run dev
```

Build:

```bash
npm run build
```

## Bluetooth signal probe (macOS)

Simple BLE probe script to inspect nearby devices (including SwitchBot-like devices), advertised service UUIDs, and optional GATT layout for a target address.

Setup:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install bleak
```

Scan nearby BLE devices:

```bash
python scripts/bt_probe.py --scan-timeout 8
```

Scan + connect to one device for GATT service/characteristic/descriptor enumeration:

```bash
python scripts/bt_probe.py --address "<BLE_ADDRESS_OR_UUID>" --scan-timeout 8 --connect-timeout 10
```

## PWA and Offline

- Manifest is generated by `vite-plugin-pwa` during build.
- Service worker is registered in `src/main.tsx`.
- The app shell and build assets are precached, so after first load it works offline.
- For full offline operation, the app now uses local/system fonts only (no remote font fetch dependency).

## Musical Model

### Separate pitch concepts

- **Reference pitch**: A4 frequency, default `440.0 Hz`
- **Base octave**: shifts the full note bank register (`c..h` and `c1..h1`)
- **Tuning system**: `Equal temperament` or `Natural / just intonation`
- **Tonal center**: selectable center (`G`, `D`, `A`, `C`, `E`)

The separation lives in `src/music/tuning.ts` and `src/store/useDroneStore.ts`.

### Note bank

- lower register: `c d e f fis g a h`
- upper register: `c1 d1 e1 f1 fis1 g1 a1 h1`

## Tuning Details

### Equal temperament

Computed from A4:

`f = A4 * 2^((midi - 69) / 12)`

### Natural / just intonation ratios

5-limit ratios referenced to C:

- `c = 1/1`
- `d = 9/8`
- `e = 5/4`
- `f = 4/3`
- `fis = 45/32`
- `g = 3/2`
- `a = 5/3`
- `h = 15/8`

For selected center:

`ratio(note, center) = normalize( ratioFromC(note) / ratioFromC(center) )`

`f = centerFrequency * ratio(note, center) * octaveMultiplier`

Natural centers for workflow presets:

- Natural G
- Natural D
- Natural A
- Natural C
- Natural E

## Audio Engine

Implemented in `src/audio/DroneEngine.ts`:

- click-free start/stop via attack/release ramps
- smooth parameter changes for gain, pan, and frequency
- per-tone gain + pan
- additive partials with per-partial:
  - ratio (harmonic or inharmonic)
  - gain (dB)
  - enable/disable
- draggable overtone bars for fast harmonic balancing
- timbre blending across `sine`, `saw`, `square`
- warm output stage: low-pass filter + compressor limiter
- built-in metronome (BPM + click volume)

## Presets

Each preset stores:

- tuning system
- tonal center
- A4 reference
- master gain
- selected tones
- per-tone gain and pan
- partial configuration
- timbre blend

Preset actions:

- load
- save active
- save as new
- duplicate
- reorder
- delete

## Extending Partials

Use UI "Add partial" or define defaults in `src/presets/defaultPresets.ts`.

Example partial:

```ts
{ id: 'p6', ratio: 7, gainDb: -26, enabled: true }
```

Inharmonic examples such as `ratio: 5.5` are valid and can produce richer bourdon colors.
